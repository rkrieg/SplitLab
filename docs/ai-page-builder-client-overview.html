AI Landing Page Builder

SplitLab — Feature 1 plan & walkthrough

What we're building

A built-in AI page builder inside SplitLab. Users describe the landing page they want in plain English, AI generates a complete page, they preview and edit it, then publish — hosted on their custom domain with tracking already wired in. Think Lovable, but native to SplitLab.

End-to-end user flow

1
Describe the page — User types a prompt (e.g. "Landing page for a personal injury firm in Miami") and picks a vertical: Lead Gen, SaaS, or Local Services.
2
AI generates the page — AI writes the copy and builds a full, unique landing page (layout, styling, sections). If the prompt is too vague, AI asks 1–2 clarifying questions first.
3
Live preview — The generated page appears in a preview panel. User sees exactly what visitors will see.
4
Edit inline — User clicks any headline, paragraph, or button text to edit it directly on the page. No drag-and-drop builder — just click and type.
5
Publish — User hits Publish. The page goes live on the client's existing custom domain (same as pasted HTML pages work today).
6
Track conversions — SplitLab automatically injects tracking. Form submissions, button clicks, and conversion goals work the same as every other SplitLab page.

What works automatically (no extra build)

Because AI pages are served the same way as custom HTML pages today, these already work out of the box:

  • Conversion tracking via tracker.js
  • Form lead capture
  • Email notifications (Resend)
  • HubSpot, Facebook, and Google integrations
  • Custom domain hosting
  • A/B testing (page can be used as a test variant)

How hosting works

Published AI pages are stored and served exactly like manually pasted HTML pages — on the client's custom domain through SplitLab's existing infrastructure. No change to how live traffic is handled.

Later (not v1): Optional dedicated subdomain hosting at pages.trysplitlab.com/your-slug for pages that don't need a custom domain.

Scope

In v1

  • AI page generation from prompt
  • Vertical selector (3 options)
  • Live preview
  • Inline text editing
  • Publish on custom domain
  • Page management (list, edit, duplicate, delete)

Coming after v1

  • UTM personalization (swap headlines/CTAs by traffic source)
  • Full page routing by UTM (different page per campaign)
  • pages.trysplitlab.com subdomain

Why this order

The AI builder is the foundation. UTM headline swapping and full page routing both depend on pages being built and stored in a structured way inside SplitLab. Ship the builder first, then layer personalization on top.
